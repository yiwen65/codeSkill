# Interview Question Bank

## Round 1: Context and boundaries
1. What core problem should this iteration solve in one sentence?
2. Who is the primary user and what outcome defines success?
3. Which platforms/environments must be supported now?
4. Which hard constraints apply (stack, infra, compliance, deadlines)?
5. What is explicitly out of scope for this iteration?

## Round 2: Functional detail
1. What is the happy-path user journey?
2. What are required inputs and outputs?
3. Which edge cases and failure modes must be handled?

## Round 3: Acceptance and tests
1. What must MVP include to be accepted?
2. Which tests are mandatory before sign-off?
3. If no tests exist, can minimal tests be added?
